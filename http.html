HTTP and HTTPS

  HTTP which stands for Hypertext Transfer Protocol, is the primary way that computers can access different websites on the internet. It is a communication protocol that allows different nodes on the network to communicate in a common language and direct nodes to the resources they are looking for. This is done through different types of requests. The primary way of retrieving resources is through a GET request. This request asks a server for some resource and the server fetches the data and returns it to the sender of the request. There are other types such as PUT, POST, DELETE, PATCH, etc. All these request methods handle a specific task and they are all used to communicate in a common language.

  HTTPS is the same as HTTP but differs in the way the communication happens between a client and a server. HTTPS stands for Hypertext Transfer Protocol Secure. This security is provided by the TLS or Transport Security Layer which communicates over port 443 instead of the standard port 80 in HTTP. Data packets are encrypted using a public key on the client and decoded on the server and sent over this persistent connection which allows secure and private communication to occur between a client and server. Each server has an SSL certificate that is signed by a Certificate Authority that allows for implicit trust in the secure connection. HTTPS is very crucial for modern day services such as online banking, credit card transactions, and many other security focused applications communicating over the internet.
